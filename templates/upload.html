How to export the call report (step-by-step)
Before you start
  • You need access to the customer's RingCentral account — typically via SCP (Service Console Portal) with the customer's authorization.
  • From there, open the analytics site at analytics.ringcentral.com for that account.
  1. In the left sidebar, open Analytics → Performance Reports.
  2. At the top, click the Calls tab (next to Queues / Users / Meetings). This is the call-by-call detail view — you'll see a “Total … Sessions” count above the table.
  3. Click the date pill at the top (it says “Yesterday” by default) and pick your range — e.g. Last Month, or use Custom Range for an exact window — then click Done.
  4. Leave the filter dropdowns at their defaults so nothing is excluded: Call Type = All Types, Queue Calls = All Calls, Call Length = All Length, Site Calls = all sites. (To focus on specific call queues, use the first dropdown → Queues tab and select them.)
  5. Click Download (top-right) and choose Excel.
  6. Upload that file as file 1 above.
  7. Then repeat for the Queues report: click the Queues tab (next to Calls), keep the same date range, Download → Excel, and upload it as file 2. This file is what carries the abandoned-in-queue counts, wait times, and service level — the Calls export doesn't include them.
The one thing that matters: the columns

Whatever the menu looks like in your account, the export just needs to contain these columns. The first five are required; the rest unlock the hourly chart, repeat-caller detection, and answered-call confirmation:

Required: Session Id · Call Direction · Result · Call Length · Queue
Recommended: Handle Time · Call Start Time · From Number

The Calls tab already includes these (From, Result, Length, Handle Time, Start Time, Queue, Direction). If you see a Select Columns option, make sure all eight are checked. If the export is missing a required column, the tool will tell you which one on the next screen.

Tip: the export is one row per call leg — this tool automatically de-duplicates those into one row per call (session), so you don't need to clean anything up first.
